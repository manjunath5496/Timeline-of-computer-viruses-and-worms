<h2>Timeline of computer viruses and worms </h2>
<h2><span id="Pre-1970" class="mw-headline">Pre-1970</span></h2>
<ul>
<li><a title="John von Neumann" href="https://en.wikipedia.org/wiki/John_von_Neumann">John von Neumann</a>'s article on the "Theory of self-reproducing automata" is published in 1966.<sup id="cite_ref-1" class="reference"></sup>&nbsp;The article is based on lectures given by von Neumann at the&nbsp;<a class="mw-redirect" title="University of Illinois" href="https://en.wikipedia.org/wiki/University_of_Illinois">University of Illinois</a>&nbsp;about the "Theory and Organization of Complicated Automata" in 1949.</li>
</ul>
<h2><span id="1971.E2.80.931975"></span><span id="1971&ndash;1975" class="mw-headline">1971&ndash;1975</span></h2>
<h3><span id="1970_.28Fiction.29"></span><span id="1970_(Fiction)" class="mw-headline">1970 (Fiction)</span></h3>
<ul>
<li>The first story written about a computer virus is&nbsp;<em>The Scarred Man</em>&nbsp;by&nbsp;<a title="Gregory Benford" href="https://en.wikipedia.org/wiki/Gregory_Benford">Gregory Benford</a>.</li>
</ul>
<h3><span id="1971" class="mw-headline">1971</span></h3>
<ul>
<li>The&nbsp;<a title="Creeper (program)" href="https://en.wikipedia.org/wiki/Creeper_(program)">Creeper system</a>, an experimental self-replicating program, is written by Bob Thomas at&nbsp;<a title="BBN Technologies" href="https://en.wikipedia.org/wiki/BBN_Technologies">BBN Technologies</a>&nbsp;to test John von Neumann's theory.<sup id="cite_ref-2" class="reference"></sup>&nbsp;Creeper infected DEC&nbsp;<a title="PDP-10" href="https://en.wikipedia.org/wiki/PDP-10">PDP-10</a>&nbsp;computers running the&nbsp;<a title="TENEX (operating system)" href="https://en.wikipedia.org/wiki/TENEX_(operating_system)">TENEX operating system</a>. Creeper gained access via the&nbsp;<a title="ARPANET" href="https://en.wikipedia.org/wiki/ARPANET">ARPANET</a>&nbsp;and copied itself to the remote system where the message "I'm the creeper, catch me if you can!" was displayed. The&nbsp;<a title="Reaper (program)" href="https://en.wikipedia.org/wiki/Reaper_(program)">Reaper</a>&nbsp;program was later created to delete Creeper.<sup id="cite_ref-3" class="reference"></sup></li>
</ul>
<h3><span id="1972_.28Fiction.29"></span><span id="1972_(Fiction)" class="mw-headline">1972 (Fiction)</span></h3>
<ul>
<li>The science fiction novel,&nbsp;<em><a title="When HARLIE Was One" href="https://en.wikipedia.org/wiki/When_HARLIE_Was_One">When HARLIE Was One</a></em>, by&nbsp;<a title="David Gerrold" href="https://en.wikipedia.org/wiki/David_Gerrold">David Gerrold</a>, contains one of the first fictional representations of a&nbsp;<a title="Computer virus" href="https://en.wikipedia.org/wiki/Computer_virus">computer virus</a>, as well as one of the first uses of the word "virus" to denote a program that infects a computer.</li>
</ul>
<h3><span id="1973_.28Fiction.29"></span><span id="1973_(Fiction)" class="mw-headline">1973 (Fiction)</span></h3>
<ul>
<li>In fiction, the 1973&nbsp;<a title="Michael Crichton" href="https://en.wikipedia.org/wiki/Michael_Crichton">Michael Crichton</a>&nbsp;movie&nbsp;<em><a title="Westworld (film)" href="https://en.wikipedia.org/wiki/Westworld_(film)">Westworld</a></em>&nbsp;made an early mention of the concept of a computer virus, being a central plot theme that causes androids to run amok.<sup id="cite_ref-4" class="reference"></sup>&nbsp;<a title="Alan Oppenheimer" href="https://en.wikipedia.org/wiki/Alan_Oppenheimer">Alan Oppenheimer</a>'s character summarizes the problem by stating that "...there's a clear pattern here which suggests an analogy to an infectious disease process, spreading from one...area to the next." To which the replies are stated: "Perhaps there are superficial similarities to disease" and, "I must confess I find it difficult to believe in a disease of machinery."<sup id="cite_ref-5" class="reference"></sup>&nbsp;(Crichton's earlier work, the 1969 novel&nbsp;<em><a title="The Andromeda Strain" href="https://en.wikipedia.org/wiki/The_Andromeda_Strain">The Andromeda Strain</a></em>&nbsp;and&nbsp;<a title="The Andromeda Strain (film)" href="https://en.wikipedia.org/wiki/The_Andromeda_Strain_(film)">1971 film</a>&nbsp;were about an extraterrestrial biological virus-like disease that threatened the human race.)</li>
</ul>
<h3><span id="1974" class="mw-headline">1974</span></h3>
<ul>
<li>The Rabbit (or Wabbit) virus, more a&nbsp;<a title="Fork bomb" href="https://en.wikipedia.org/wiki/Fork_bomb">fork bomb</a>&nbsp;than a virus, is written. The Rabbit virus makes multiple copies of itself on a single computer (and was named "<a title="Rabbit" href="https://en.wikipedia.org/wiki/Rabbit">Rabbit</a>" for the speed at which it did so) until it clogs the system, reducing system performance, before finally reaching a threshold and crashing the computer.<sup id="cite_ref-6" class="reference"></sup></li>
</ul>
<h3><span id="1975" class="mw-headline">1975</span></h3>
<ul>
<li>April: ANIMAL is written by&nbsp;<a title="John Walker (programmer)" href="https://en.wikipedia.org/wiki/John_Walker_(programmer)">John Walker</a>&nbsp;for the&nbsp;<a title="UNIVAC 1100/2200 series" href="https://en.wikipedia.org/wiki/UNIVAC_1100/2200_series">UNIVAC 1108</a>.<sup id="cite_ref-http://www.fourmilab.ch/documents/univac/animalsrc.html_7-0" class="reference"></sup>&nbsp;ANIMAL asked a number of questions of the user in an attempt to guess the type of animal that the user was thinking of, while the related program PERVADE would create a copy of itself and ANIMAL in every directory to which the current user had access. It spread across the multi-user UNIVACs when users with overlapping permissions discovered the game, and to other computers when tapes were shared. The program was carefully written to avoid damage to existing file or directory structures, and not to copy itself if permissions did not exist or if damage could result. Its spread was therefore halted by an OS upgrade which changed the format of the file status tables that PERVADE used for safe copying. Though non-malicious, "Pervading Animal" represents the first&nbsp;<a title="Trojan horse (computing)" href="https://en.wikipedia.org/wiki/Trojan_horse_(computing)">Trojan</a>&nbsp;"in the wild".<sup id="cite_ref-http://www.fourmilab.ch/documents/univac/animal.html_8-0" class="reference"></sup></li>
<li>The novel&nbsp;<em><a title="The Shockwave Rider" href="https://en.wikipedia.org/wiki/The_Shockwave_Rider">The Shockwave Rider</a></em>&nbsp;by&nbsp;<a title="John Brunner (novelist)" href="https://en.wikipedia.org/wiki/John_Brunner_(novelist)">John Brunner</a>&nbsp;is published, coining the word "<a title="Computer worm" href="https://en.wikipedia.org/wiki/Computer_worm">worm</a>" to describe a program that propagates itself through a&nbsp;<a title="Computer network" href="https://en.wikipedia.org/wiki/Computer_network">computer network</a>.<sup id="cite_ref-ClassicSci-FiReviews_9-0" class="reference"></sup></li>
</ul>
<h2><span id="1981.E2.80.931989"></span><span id="1981&ndash;1989" class="mw-headline">1981&ndash;1989</span></h2>
<h3><span id="1981" class="mw-headline">1981</span></h3>
<ul>
<li>A program called&nbsp;<a title="Elk Cloner" href="https://en.wikipedia.org/wiki/Elk_Cloner">Elk Cloner</a>, written for&nbsp;<a title="Apple II series" href="https://en.wikipedia.org/wiki/Apple_II_series">Apple II</a>&nbsp;systems, was created by high school student&nbsp;<a title="Rich Skrenta" href="https://en.wikipedia.org/wiki/Rich_Skrenta">Richard Skrenta</a>, originally as a prank. The Apple II was particularly vulnerable due to the storage of its operating system computer virus outbreak in history.<sup id="cite_ref-10" class="reference"></sup></li>
</ul>
<h3><span id="1983" class="mw-headline">1983</span></h3>
<ul>
<li>November: The term "virus" is re-coined by&nbsp;<a title="Fred Cohen" href="https://en.wikipedia.org/wiki/Fred_Cohen">Frederick B. Cohen</a>&nbsp;in describing self-replicating computer programs. In 1984 Cohen uses the phrase "computer virus" (suggested by his teacher&nbsp;<a title="Leonard Adleman" href="https://en.wikipedia.org/wiki/Leonard_Adleman">Leonard Adleman</a>) to describe the operation of such programs in terms of "infection". He defines a "virus" as "a program that can 'infect' other programs by modifying them to include a possibly evolved copy of itself." Cohen demonstrates a virus-like program on a&nbsp;<a title="VAX" href="https://en.wikipedia.org/wiki/VAX">VAX11/750</a>&nbsp;system at&nbsp;<a title="Lehigh University" href="https://en.wikipedia.org/wiki/Lehigh_University">Lehigh University</a>. The program could install itself in, or infect, other system objects.</li>
</ul>
<h3><span id="1984" class="mw-headline">1984</span></h3>
<ul>
<li>August:&nbsp;<a title="Ken Thompson" href="https://en.wikipedia.org/wiki/Ken_Thompson">Ken Thompson</a>&nbsp;publishes his seminal paper,&nbsp;<em><a class="mw-redirect" title="Reflections on Trusting Trust" href="https://en.wikipedia.org/wiki/Reflections_on_Trusting_Trust">Reflections on Trusting Trust</a></em>, in which he describes how he modified a&nbsp;<a title="C (programming language)" href="https://en.wikipedia.org/wiki/C_(programming_language)">C</a>&nbsp;<a title="Compiler" href="https://en.wikipedia.org/wiki/Compiler">compiler</a>&nbsp;so that when used to compile a specific version of the&nbsp;<a title="Unix" href="https://en.wikipedia.org/wiki/Unix">Unix</a>&nbsp;operating system, it inserts a&nbsp;<a title="Backdoor (computing)" href="https://en.wikipedia.org/wiki/Backdoor_(computing)">backdoor</a>&nbsp;into the&nbsp;<a title="Login" href="https://en.wikipedia.org/wiki/Login">login</a>&nbsp;command, and when used to compile a new copy of itself, it inserts the backdoor insertion code, even if neither the backdoor nor the backdoor insertion code is present in the&nbsp;<a title="Source code" href="https://en.wikipedia.org/wiki/Source_code">source code</a>&nbsp;of this new copy.<sup id="cite_ref-12" class="reference"></sup></li>
</ul>
<h3><span id="1986" class="mw-headline">1986</span></h3>
<ul>
<li>January: The&nbsp;<a title="Brain (computer virus)" href="https://en.wikipedia.org/wiki/Brain_(computer_virus)">Brain</a>&nbsp;<a title="Boot sector" href="https://en.wikipedia.org/wiki/Boot_sector">boot sector</a>&nbsp;virus is released. Brain is considered the first&nbsp;<a title="IBM PC compatible" href="https://en.wikipedia.org/wiki/IBM_PC_compatible">IBM PC compatible</a>&nbsp;virus, and the program responsible for the first IBM PC compatible virus epidemic. The virus is also known as Lahore, Pakistani, Pakistani Brain, and Pakistani flu as it was created in&nbsp;<a title="Lahore" href="https://en.wikipedia.org/wiki/Lahore">Lahore</a>,&nbsp;<a title="Pakistan" href="https://en.wikipedia.org/wiki/Pakistan">Pakistan</a>&nbsp;by 19-year-old Pakistani programmer, Basit Farooq Alvi, and his brother,&nbsp;<a class="mw-redirect" title="Amjad Farooq Alvi" href="https://en.wikipedia.org/wiki/Amjad_Farooq_Alvi">Amjad Farooq Alvi</a>.<sup id="cite_ref-theregisterJanuary2006_13-0" class="reference"></sup></li>
<li>December: Ralf Burger presented the Virdem model of programs at a meeting of the underground&nbsp;<a title="Chaos Computer Club" href="https://en.wikipedia.org/wiki/Chaos_Computer_Club">Chaos Computer Club</a>&nbsp;in Germany. The Virdem model represented the first programs that could replicate themselves via addition of their code to executable DOS files in COM format.<sup id="cite_ref-TAOCVRAD-Szor_14-0" class="reference"></sup></li>
</ul>
<h3><span id="1987" class="mw-headline">1987</span></h3>
<ul>
<li>Appearance of the Vienna virus, which was subsequently neutralized &ndash; the first time this had happened on the&nbsp;<a title="IBM" href="https://en.wikipedia.org/wiki/IBM">IBM</a>&nbsp;platform.<sup id="cite_ref-wentworth_15-0" class="reference"></sup></li>
<li>Appearance of Lehigh virus (discovered at its&nbsp;<a title="Lehigh University" href="https://en.wikipedia.org/wiki/Lehigh_University">namesake university</a>),<sup id="cite_ref-wentworth_15-1" class="reference"></sup>&nbsp;boot sector viruses such as Yale from US,&nbsp;<a title="Stoned (computer virus)" href="https://en.wikipedia.org/wiki/Stoned_(computer_virus)">Stoned</a>&nbsp;from New Zealand,&nbsp;<a title="Ping-Pong virus" href="https://en.wikipedia.org/wiki/Ping-Pong_virus">Ping Pong</a>&nbsp;from Italy, and appearance of first self-encrypting file virus,&nbsp;<a title="Cascade (computer virus)" href="https://en.wikipedia.org/wiki/Cascade_(computer_virus)">Cascade</a>. Lehigh was stopped on campus before it spread to the "wild" (to computers beyond the university), and has never been found elsewhere as a result. A subsequent infection of Cascade in the offices of IBM Belgium led to IBM responding with its own antivirus product development. Prior to this, antivirus solutions developed at IBM were intended for staff use only.</li>
<li>October: The&nbsp;<a title="Jerusalem (computer virus)" href="https://en.wikipedia.org/wiki/Jerusalem_(computer_virus)">Jerusalem virus</a>, part of the (at that time unknown) Suriv family, is detected in the city of&nbsp;<a title="Jerusalem" href="https://en.wikipedia.org/wiki/Jerusalem">Jerusalem</a>. The virus destroys all executable files on infected machines upon every occurrence of Friday the 13th (except Friday 13 November 1987 making its first trigger date May 13, 1988). Jerusalem caused a worldwide epidemic in 1988.<sup id="cite_ref-wentworth_15-2" class="reference"></sup></li>
<li>November: The&nbsp;<a title="SCA (computer virus)" href="https://en.wikipedia.org/wiki/SCA_(computer_virus)">SCA virus</a>, a boot sector virus for&nbsp;<a title="Amiga" href="https://en.wikipedia.org/wiki/Amiga">Amiga</a>&nbsp;computers, appears. It immediately creates a pandemic virus-writer storm. A short time later,&nbsp;<a class="mw-redirect" title="Swiss Cracking Association" href="https://en.wikipedia.org/wiki/Swiss_Cracking_Association">SCA</a>&nbsp;releases another, considerably more destructive virus, the&nbsp;<a title="Byte Bandit" href="https://en.wikipedia.org/wiki/Byte_Bandit">Byte Bandit</a>.</li>
<li>December:&nbsp;<a title="Christmas Tree EXEC" href="https://en.wikipedia.org/wiki/Christmas_Tree_EXEC">Christmas Tree EXEC</a>&nbsp;was the first widely disruptive replicating network program, which paralyzed several international computer networks in December 1987. It was written in&nbsp;<a title="Rexx" href="https://en.wikipedia.org/wiki/Rexx">Rexx</a>&nbsp;on the&nbsp;<a title="VM (operating system)" href="https://en.wikipedia.org/wiki/VM_(operating_system)">VM/CMS</a>&nbsp;operating system and originated in what was then&nbsp;<a title="West Germany" href="https://en.wikipedia.org/wiki/West_Germany">West Germany</a>. It re-emerged in 1990.</li>
</ul>
<h3><span id="1988" class="mw-headline">1988</span></h3>
<ul>
<li>March 1: The&nbsp;<a title="Ping-Pong virus" href="https://en.wikipedia.org/wiki/Ping-Pong_virus">Ping-Pong virus</a>&nbsp;(also called Boot, Bouncing Ball, Bouncing Dot, Italian, Italian-A or VeraCruz), an MS-DOS boot sector virus, is discovered at the&nbsp;<a title="University of Turin" href="https://en.wikipedia.org/wiki/University_of_Turin">University of Turin</a>&nbsp;in Italy.</li>
<li>June: The&nbsp;<a title="Festering Hate" href="https://en.wikipedia.org/wiki/Festering_Hate">CyberAIDS and Festering Hate</a>&nbsp;<a title="Apple ProDOS" href="https://en.wikipedia.org/wiki/Apple_ProDOS">Apple ProDOS</a>&nbsp;viruses spreads from underground pirate BBS systems and starts infecting mainstream networks. Festering Hate was the last iteration of the CyberAIDS series extending back to 1985 and 1986. Unlike the few Apple viruses that had come before which were essentially annoying, but did no damage, the Festering Hate series of viruses was extremely destructive, spreading to all system files it could find on the host computer (hard drive, floppy, and system memory) and then destroying everything when it could no longer find any uninfected files.</li>
<li>November 2: The&nbsp;<a title="Morris worm" href="https://en.wikipedia.org/wiki/Morris_worm">Morris worm</a>, created by&nbsp;<a title="Robert Tappan Morris" href="https://en.wikipedia.org/wiki/Robert_Tappan_Morris">Robert Tappan Morris</a>, infects&nbsp;<a title="Digital Equipment Corporation" href="https://en.wikipedia.org/wiki/Digital_Equipment_Corporation">DEC</a>&nbsp;<a title="VAX" href="https://en.wikipedia.org/wiki/VAX">VAX</a>&nbsp;and&nbsp;<a title="Sun Microsystems" href="https://en.wikipedia.org/wiki/Sun_Microsystems">Sun</a>&nbsp;machines running&nbsp;<a title="Berkeley Software Distribution" href="https://en.wikipedia.org/wiki/Berkeley_Software_Distribution">BSD UNIX</a>&nbsp;that are connected to the&nbsp;<a title="Internet" href="https://en.wikipedia.org/wiki/Internet">Internet</a>, and becomes the first worm to spread extensively "in the wild", and one of the first well-known programs exploiting&nbsp;<a title="Buffer overflow" href="https://en.wikipedia.org/wiki/Buffer_overflow">buffer overrun</a>&nbsp;vulnerabilities.</li>
<li>December: The&nbsp;<a title="Father Christmas (computer worm)" href="https://en.wikipedia.org/wiki/Father_Christmas_(computer_worm)">Father Christmas worm</a>&nbsp;attacks&nbsp;<a title="Digital Equipment Corporation" href="https://en.wikipedia.org/wiki/Digital_Equipment_Corporation">DEC</a>&nbsp;<a title="VAX" href="https://en.wikipedia.org/wiki/VAX">VAX</a>&nbsp;machines running&nbsp;<a class="mw-redirect" title="VAX/VMS" href="https://en.wikipedia.org/wiki/VAX/VMS">VAX/VMS</a>&nbsp;that are connected to the&nbsp;<a title="DECnet" href="https://en.wikipedia.org/wiki/DECnet#The_DECnet_Internet">DECnet Internet</a>&nbsp;(an international scientific research network using&nbsp;<a title="DECnet" href="https://en.wikipedia.org/wiki/DECnet">DECnet</a>&nbsp;protocols), affecting&nbsp;<a title="NASA" href="https://en.wikipedia.org/wiki/NASA">NASA</a>&nbsp;and other research centers. Its purpose was to deliver a Christmas greeting to all affected users.</li>
</ul>
<h3><span id="1989" class="mw-headline">1989</span></h3>
<ul>
<li>October:&nbsp;<a class="mw-redirect" title="Ghostball" href="https://en.wikipedia.org/wiki/Ghostball">Ghostball</a>, the first&nbsp;<a class="new" title="Multipartite virus (page does not exist)" href="https://en.wikipedia.org/w/index.php?title=Multipartite_virus&amp;action=edit&amp;redlink=1">multipartite virus</a>, is discovered by&nbsp;<a class="mw-redirect" title="Fri&eth;rik Sk&uacute;lason" href="https://en.wikipedia.org/wiki/Fri%C3%B0rik_Sk%C3%BAlason">Fri&eth;rik Sk&uacute;lason</a>. It infects both executable .COM-files and boot sectors on MS-DOS systems.</li>
<li>December: Several thousand floppy disks containing the&nbsp;<a title="AIDS (Trojan horse)" href="https://en.wikipedia.org/wiki/AIDS_(Trojan_horse)">AIDS</a>&nbsp;Trojan, the first known&nbsp;<a title="Ransomware" href="https://en.wikipedia.org/wiki/Ransomware">ransomware</a>, are mailed to subscribers of PC Business World magazine and a WHO AIDS conference mailing list. This DOS Trojan lies dormant for 90 boot cycles, then encrypts all filenames on the system, displaying a notice asking for $189 to be sent to a post office box in Panama in order to receive a decryption program.</li>
</ul>

Chameleon series debuted with the release of&nbsp;<a title="1260 (computer virus)" href="https://en.wikipedia.org/wiki/1260_(computer_virus)">1260</a>.<sup id="cite_ref-16" class="reference"></sup><sup id="cite_ref-17" class="reference"></sup><sup id="cite_ref-18" class="reference"></sup></li>

Later assessments of the damage showed the aftermath to be minimal.&nbsp;<a title="John McAfee" href="https://en.wikipedia.org/wiki/John_McAfee">John McAfee</a>&nbsp;had been quoted by the media as saying that 5 million computers would be affected. </ul>

<h2><span id="2000.E2.80.932009"></span><span id="2000&ndash;2009" class="mw-headline">2000&ndash;2009</span></h2>
<h3><span id="2000" class="mw-headline">2000</span></h3>
<ul>

<li>Summer 2011: The Morto worm attempts to propagate itself to additional computers via the Microsoft Windows&nbsp;<a title="Remote Desktop Protocol" href="https://en.wikipedia.org/wiki/Remote_Desktop_Protocol">Remote Desktop Protocol</a>&nbsp;(RDP). Morto spreads by forcing infected systems to scan for Windows servers allowing RDP login. Once Morto finds an RDP-accessible system, it attempts to log into a domain or local system account named 'Administrator' using a number of common passwords.<sup id="cite_ref-morto_65-0" class="reference"></sup>&nbsp;A detailed overview of how the worm works &ndash; along with the password dictionary Morto uses &ndash; was done by&nbsp;<a title="Imperva" href="https://en.wikipedia.org/wiki/Imperva">Imperva</a>.<sup id="cite_ref-imperva_66-0" class="reference"></sup></li>
<li>July 13: the&nbsp;<a class="mw-redirect" title="ZeroAccess" href="https://en.wikipedia.org/wiki/ZeroAccess">ZeroAccess</a>&nbsp;rootkit (also known as Sirefef or max++) was discovered.</li>
<li>September 1:&nbsp;<a title="Duqu" href="https://en.wikipedia.org/wiki/Duqu">Duqu</a>&nbsp;is a worm thought to be related to the&nbsp;<a title="Stuxnet" href="https://en.wikipedia.org/wiki/Stuxnet">Stuxnet</a>&nbsp;worm. The Laboratory of Cryptography and System Security (<a title="CrySyS Lab" href="https://en.wikipedia.org/wiki/CrySyS_Lab">CrySyS Lab</a>)<sup id="cite_ref-67" class="reference"></sup>&nbsp;of the&nbsp;<a title="Budapest University of Technology and Economics" href="https://en.wikipedia.org/wiki/Budapest_University_of_Technology_and_Economics">Budapest University of Technology and Economics</a>&nbsp;in&nbsp;<a title="Hungary" href="https://en.wikipedia.org/wiki/Hungary">Hungary</a>&nbsp;discovered the threat, analysed the malware, and wrote a 60-page report naming the threat Duqu.<sup id="cite_ref-68" class="reference"></sup><sup id="cite_ref-69" class="reference"></sup>&nbsp;Duqu gets its name from the prefix "~DQ" it gives to the names of files it creates.<sup id="cite_ref-syamantecduqu_70-0" class="reference"></sup></li>
</ul>
<h3><span id="2012" class="mw-headline">2012</span></h3>
<ul>
<li>May:&nbsp;<a title="Flame (malware)" href="https://en.wikipedia.org/wiki/Flame_(malware)">Flame</a>&nbsp;&ndash; also known as Flamer, sKyWIper, and Skywiper &ndash; a modular computer malware that attacks computers running Microsoft Windows. Used for targeted&nbsp;<a class="mw-redirect" title="Cyber espionage" href="https://en.wikipedia.org/wiki/Cyber_espionage">cyber espionage</a>&nbsp;in Middle Eastern countries. Its discovery was announced on 28 May 2012 by MAHER Center of Iranian National Computer Emergency Response Team (CERT), Kaspersky Lab and CrySyS Lab of the Budapest University of Technology and Economics. CrySyS stated in their report that "sKyWIper is certainly the most sophisticated malware we encountered during our practice; arguably, it is the most complex malware ever found".<sup id="cite_ref-:0_71-0" class="reference"></sup></li>
<li>August 16:&nbsp;<a title="Shamoon" href="https://en.wikipedia.org/wiki/Shamoon">Shamoon</a>&nbsp;is a computer virus designed to target computers running Microsoft Windows in the energy sector. Symantec, Kaspersky Lab, and Seculert announced its discovery on August 16, 2012.</li>
<li>September 20: NGRBot is a worm that uses the IRC network for file transfer, sending and receiving commands between zombie network machines and the attacker's IRC server, and monitoring and controlling network connectivity and intercept. It employs a user-mode rootkit technique to hide and steal its victim's information. This family of bot is also designed to infect HTML pages with inline frames (<a title="HTML element" href="https://en.wikipedia.org/wiki/HTML_element#Frames">iframes</a>), causing redirections, blocking victims from getting updates from security/antimalware products, and killing those services. The bot is designed to connect via a predefined IRC channel and communicate with a remote botnet.<sup id="cite_ref-:1_72-0" class="reference"></sup><sup id="cite_ref-:2_73-0" class="reference"></sup></li>
</ul>
<h3><span id="2013" class="mw-headline">2013</span></h3>
<ul>
<li>September: The&nbsp;<a title="CryptoLocker" href="https://en.wikipedia.org/wiki/CryptoLocker">CryptoLocker</a>&nbsp;Trojan horse is discovered. CryptoLocker encrypts the files on a user's hard drive, then prompts them to pay a ransom to the developer in order to receive the decryption key. In the following months, a number of copycat ransomware Trojans were also discovered.</li>
<li>December: The&nbsp;<a title="Gameover ZeuS" href="https://en.wikipedia.org/wiki/Gameover_ZeuS">Gameover ZeuS</a>&nbsp;Trojan is discovered. This type of virus steals one's login details on popular Web sites that involve monetary transactions. It works by detecting a login page, then proceeds to inject a malicious code into the page,&nbsp;<a title="Keystroke logging" href="https://en.wikipedia.org/wiki/Keystroke_logging">keystroke logging</a>&nbsp;the computer user's details.</li>
<li>December:&nbsp;<a title="Linux.Darlloz" href="https://en.wikipedia.org/wiki/Linux.Darlloz">Linux.Darlloz</a>&nbsp;targets the&nbsp;<a title="Internet of things" href="https://en.wikipedia.org/wiki/Internet_of_things">Internet of things</a>&nbsp;and infects&nbsp;<a title="Router (computing)" href="https://en.wikipedia.org/wiki/Router_(computing)">routers</a>,&nbsp;<a class="mw-redirect" title="Security camera" href="https://en.wikipedia.org/wiki/Security_camera">security cameras</a>,&nbsp;<a title="Set-top box" href="https://en.wikipedia.org/wiki/Set-top_box">set-top boxes</a>&nbsp;by exploiting a&nbsp;<a title="PHP" href="https://en.wikipedia.org/wiki/PHP">PHP</a>&nbsp;vulnerability.<sup id="cite_ref-:3_74-0" class="reference"></sup><sup id="cite_ref-:4_75-0" class="reference"></sup></li>
</ul>
<h3><span id="2014" class="mw-headline">2014</span></h3>

Regin is a&nbsp;<a title="Dropper (malware)" href="https://en.wikipedia.org/wiki/Dropper_(malware)">dropper</a>&nbsp;that is primarily spread via spoofed Web pages. Once downloaded, Regin quietly downloads extensions of itself, making it difficult to be detected via anti-virus signatures. It is suspected to have been created by the United States and United Kingdom over a period of months or years, as a tool for espionage and mass surveillance.</li>

<h3><span id="2016" class="mw-headline">2016</span></h3>
<ul>
<li>January: A trojan named "<a class="mw-redirect" title="MEMZ trojan" href="https://en.wikipedia.org/wiki/MEMZ_trojan">MEMZ</a>" is created. The creator, Leurak, explained that the trojan was intended merely as a joke.<sup id="cite_ref-memz_1_80-0" class="reference"></sup>&nbsp;The trojan alerts the user to the fact that it is a trojan and warns them that if they proceed, the computer may no longer be usable. It contains complex payloads that corrupt the system, displaying&nbsp;<a title="Visual artifact" href="https://en.wikipedia.org/wiki/Visual_artifact">artifacts</a>&nbsp;on the screen as it runs. Once run, the application cannot be closed without causing further damage to the computer, which will stop functioning properly regardless. When the computer is restarted, in place of the&nbsp;<a title="Bootsplash" href="https://en.wikipedia.org/wiki/Bootsplash">bootsplash</a>&nbsp;is a message that reads "Your computer has been trashed by the MEMZ Trojan. Now enjoy the Nyan cat&hellip;", which follows with an animation of the&nbsp;<a title="Nyan Cat" href="https://en.wikipedia.org/wiki/Nyan_Cat">Nyan Cat</a>.<sup id="cite_ref-memz_2_81-0" class="reference"></sup></li>
<li>February: Ransomware&nbsp;<a title="Locky" href="https://en.wikipedia.org/wiki/Locky">Locky</a>&nbsp;with its over 60 derivatives spread throughout Europe and infected several million computers. At the height of the spread over five thousand computers per hour were infected in Germany alone.<sup id="cite_ref-:7_82-0" class="reference"></sup>&nbsp;Although&nbsp;<a title="Ransomware" href="https://en.wikipedia.org/wiki/Ransomware">ransomware</a>&nbsp;was not a new thing at the time, insufficient cyber security as well as a lack of standards in IT was responsible for the high number of infections.<sup id="cite_ref-:8_83-0" class="reference"></sup>&nbsp;Unfortunately, even up to date antivirus and internet security software was unable to protect systems from early versions of Locky.<sup id="cite_ref-:9_84-0" class="reference"></sup></li>
<li>February:&nbsp;<a title="Tiny Banker Trojan" href="https://en.wikipedia.org/wiki/Tiny_Banker_Trojan">Tiny Banker Trojan</a>&nbsp;(Tinba) makes headlines.<sup id="cite_ref-:10_85-0" class="reference"></sup>&nbsp;Since its discovery, it has been found to have infected more than two dozen major banking institutions in the United States, including TD Bank, Chase, HSBC, Wells Fargo, PNC and Bank of America.<sup id="cite_ref-:11_86-0" class="reference"></sup><sup id="cite_ref-:12_87-0" class="reference"></sup>&nbsp;Tiny Banker Trojan uses HTTP injection to force the user's computer to believe that it is on the bank's website. This spoof page will look and function just as the real one. The user then enters their information to log on, at which point Tinba can launch the bank webpage's "incorrect login information" return, and redirect the user to the real website. This is to trick the user into thinking they had entered the wrong information and proceed as normal, although now Tinba has captured the credentials and sent them to its host.<sup id="cite_ref-:13_88-0" class="reference"></sup><sup id="cite_ref-:14_89-0" class="reference"></sup></li>
<li>September:&nbsp;<a title="Mirai (malware)" href="https://en.wikipedia.org/wiki/Mirai_(malware)">Mirai</a>&nbsp;creates headlines by launching some of the most powerful and disruptive DDoS attacks seen to date by infecting the&nbsp;<a class="mw-redirect" title="Internet of Things" href="https://en.wikipedia.org/wiki/Internet_of_Things">Internet of Things</a>. Mirai ends up being used in the DDoS attack on 20 September 2016 on the&nbsp;<a class="mw-redirect" title="Krebs on Security" href="https://en.wikipedia.org/wiki/Krebs_on_Security">Krebs on Security</a>&nbsp;site which reached 620 Gbit/s.<sup id="cite_ref-:15_90-0" class="reference"></sup>&nbsp;<a title="Ars Technica" href="https://en.wikipedia.org/wiki/Ars_Technica">Ars Technica</a>&nbsp;also reported a 1 Tbit/s attack on French web host&nbsp;<a title="OVH" href="https://en.wikipedia.org/wiki/OVH">OVH</a>.<sup id="cite_ref-securityintelligence_91-0" class="reference"></sup>&nbsp;On 21 October 2016 multiple major DDoS attacks in&nbsp;<a class="mw-redirect" title="DNS" href="https://en.wikipedia.org/wiki/DNS">DNS</a>&nbsp;services of DNS service provider&nbsp;<a title="Dyn (company)" href="https://en.wikipedia.org/wiki/Dyn_(company)">Dyn</a>&nbsp;occurred using Mirai malware installed on a large number of&nbsp;<a title="Internet of things" href="https://en.wikipedia.org/wiki/Internet_of_things">IoT devices</a>, resulting in the inaccessibility of several high-profile websites such as&nbsp;<a title="GitHub" href="https://en.wikipedia.org/wiki/GitHub">GitHub</a>,&nbsp;<a title="Twitter" href="https://en.wikipedia.org/wiki/Twitter">Twitter</a>,&nbsp;<a title="Reddit" href="https://en.wikipedia.org/wiki/Reddit">Reddit</a>,&nbsp;<a title="Netflix" href="https://en.wikipedia.org/wiki/Netflix">Netflix</a>,&nbsp;<a title="Airbnb" href="https://en.wikipedia.org/wiki/Airbnb">Airbnb</a>&nbsp;and many others.<sup id="cite_ref-:16_92-0" class="reference"></sup>&nbsp;The attribution of the attack to the Mirai botnet was originally reported by BackConnect Inc., a security firm.<sup id="cite_ref-:17_93-0" class="reference"></sup></li>
</ul>
<h3><span id="2017" class="mw-headline">2017</span></h3>
<ul>
<li>May: The&nbsp;<a title="WannaCry ransomware attack" href="https://en.wikipedia.org/wiki/WannaCry_ransomware_attack">WannaCry ransomware attack</a>&nbsp;spreads globally. Exploits revealed in the&nbsp;<a class="mw-redirect" title="NSA" href="https://en.wikipedia.org/wiki/NSA">NSA</a>&nbsp;hacking toolkit leak of late 2016 were used to enable the propagation of the malware.<sup id="cite_ref-:18_94-0" class="reference"></sup>&nbsp;Shortly after the news of the infections broke online, a UK cybersecurity researcher in collaboration with others found and activated a "kill switch" hidden within the ransomware, effectively halting the initial wave of its global propagation.<sup id="cite_ref-:19_95-0" class="reference"></sup>&nbsp;The next day, researchers announced that they had found new variants of the malware without the kill switch.<sup id="cite_ref-:20_96-0" class="reference"></sup></li>
<li>June: The&nbsp;<a title="Petya (malware)" href="https://en.wikipedia.org/wiki/Petya_(malware)">Petya (malware)</a>&nbsp;attack spreads globally affecting Windows systems. Researchers at Symantec reveal that this ransomware uses the EternalBlue exploit, similar to the one used in the&nbsp;<a title="WannaCry ransomware attack" href="https://en.wikipedia.org/wiki/WannaCry_ransomware_attack">WannaCry ransomware attack</a>.<sup id="cite_ref-:21_97-0" class="reference"></sup><sup id="cite_ref-:22_98-0" class="reference"></sup><sup id="cite_ref-:23_99-0" class="reference"></sup></li>
<li>September: The&nbsp;<a title="Xafecopy Trojan" href="https://en.wikipedia.org/wiki/Xafecopy_Trojan">Xafecopy Trojan</a>&nbsp;attacks 47 countries, affecting only Android operating systems. Kaspersky Lab identified it as a malware from the Ubsod family, stealing money through click based WAP billing systems.<sup id="cite_ref-:24_100-0" class="reference"></sup><sup id="cite_ref-:25_101-0" class="reference"></sup></li>
<li>September: A new variety of Remote Access Trojan (RAT), Kedi RAT, is distributed in a Spear Phishing Campaign. The attack targeted Citrix users. The Trojan was able to evade usual system scanners. Kedi Trojan had all the characteristics of a common&nbsp;<a class="mw-redirect" title="Remote Access Trojan" href="https://en.wikipedia.org/wiki/Remote_Access_Trojan">Remote Access Trojan</a>&nbsp;and it could communicate to its Command and Control center via Gmail using common HTML, HTTP protocols.<sup id="cite_ref-:26_102-0" class="reference"></sup><sup id="cite_ref-:27_103-0" class="reference"></sup></li>
</ul>
